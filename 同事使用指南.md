# ScreenDemo v2.2.0 同事使用指南

> 本文档分为两部分：
> - **Part A（管理员）**：飞书机器人的完整搭建配置（只需一人操作一次）
> - **Part B（所有人）**：安装 ScreenDemo 并完成个人设置

---

## Part A：飞书机器人搭建（管理员操作）

> 只需一位管理员在飞书开放平台完成以下配置，之后将 App ID 和 App Secret 分发给同事即可。

### A-1. 创建企业自建应用

1. 用**企业管理员账号**访问：https://open.feishu.cn/app
2. 点击「**创建企业自建应用**」
3. 填写应用名称（例如：`ScreenDemo Bot`）、描述，上传图标（可跳过）
4. 点击「确定」，进入应用详情页

### A-2. 获取凭证（分发给同事的核心信息）

1. 左侧菜单：**基础信息 → 凭证与基础信息**
2. 记录以下两个值，后续分发给每位同事：

   | 字段 | 说明 |
   |------|------|
   | **App ID** | 格式如 `cli_a1b2c3d4e5f60001` |
   | **App Secret** | 点击「查看」复制完整密钥 |

### A-3. 启用机器人能力

1. 左侧菜单：**添加应用能力 → 机器人**
2. 点击「**添加**」，确认启用机器人

### A-4. 配置权限

1. 左侧菜单：**权限管理**
2. 搜索并开通以下权限（点击「开通权限」）：

   | 权限标识 | 说明 |
   |---------|------|
   | `im:message` | 获取与发送消息 |
   | `im:message:send_as_bot` | 以机器人身份发消息 |
   | `contact:user.base:readonly` | 读取用户基本信息（可选，用于获取 User ID）|

### A-5. 配置事件订阅

> ⚠️ **重要**：此步骤需要在本机已成功运行 ScreenDemo 且飞书连接建立后才能保存。具体时机见 Part B Step 4。

1. 左侧菜单：**事件与回调 → 回调配置**
2. 点击「编辑订阅方式」，选择「**使用长连接接收事件**」
3. 点击「**添加事件**」，搜索并勾选：
   - `im.message.receive_v1`（接收消息）
4. 暂不点击「保存」，等 Part B 完成后再保存

### A-6. 发布应用

1. 左侧菜单：**应用发布 → 版本管理与发布**
2. 点击「**创建版本**」
3. 填写版本号（如 `1.0.0`）和更新说明
4. 点击「申请发布」或「直接发布」（取决于企业设置）
5. 等待审批通过后，企业所有成员即可使用该机器人

> 💡 **提示**：审批期间，可在「开发版本」中将自己加为测试成员先行使用。

## Part B：安装与配置（每位同事操作）

### B-1. 安装 ScreenDemo

**方式一：安装包（推荐）**
1. 双击 `ScreenDemo Setup 2.2.0.exe`
2. 按安装向导完成安装
3. 桌面和开始菜单会创建快捷方式

**方式二：便携版（无需安装）**
1. 将 `ScreenDemo_Portable_2.2.0.exe` 放到任意目录
2. 双击直接运行

### B-2. 首次启动

1. 运行 ScreenDemo，程序会**最小化到系统托盘**（右下角任务栏通知区域）
2. 找到托盘图标

   ![托盘图标位置示意](隐藏在右下角时钟左侧)

3. **右键**托盘图标，菜单包含：
   - `Settings`（设置）
   - `Open Memo`（打开记录窗口）
   - `Quit`（退出）

### B-3. 获取自己的 User ID

> User ID 用于接收机器人的个人推送消息。每人的 User ID 不同，需要自行查找。

**方法一：通过机器人获取（推荐）**
1. 在飞书客户端中搜索机器人名称（如 `ScreenDemo Bot`）并打开对话
2. 发送任意一条消息（如：`id`）
3. ScreenDemo 日志或后台会打印发送方的 User ID（见下方注意事项）

> 📌 当前版本中，ScreenDemo 在收到飞书消息时会在日志中打印 User ID。
> 若要更方便地获取，可请管理员在飞书管理后台帮你查询。

**方法二：通过飞书管理后台获取（需管理员协助）**
1. 管理员访问：https://feishu.cn/admin/
2. 左侧菜单：**组织架构 → 成员与部门**
3. 搜索你的名字，点击查看详情
4. 复制「**用户 ID**」字段（格式如 `7g2aac8g`）

### B-4. 配置 ScreenDemo 飞书设置

1. 右键托盘图标 → **Settings**
2. 滚动到「**Feishu Integration**」部分
3. 按下表填写：

   | 字段 | 填写内容 | 说明 |
   |------|----------|------|
   | Enable Feishu Integration | ☑ 勾选 | 启用飞书功能 |
   | **App ID** | `管理员提供的 App ID` | 示例：`cli_a1b2c3d4e5f60001` |
   | **App Secret** | `管理员提供的 App Secret` | 保密，不要外传 |
   | **Target User ID** | `你自己的 User ID` | 用于接收推送，见 B-3 |
   | Enable Receive Messages | ☑ 勾选 | 接收飞书消息并创建记录 |
   | Enable Auto Reply | ☑ 勾选（可选） | 收到消息后自动回复确认 |
   | Enable Scheduled Push | ☑ 勾选（可选） | 定时推送未完成任务到飞书 |
   | Daily Push Times | `09:00` / `18:00` | 自定义每日推送时间 |

4. 点击「**Save Settings**」

5. 等待约 5 秒，ScreenDemo 会在后台建立与飞书的长连接。
   连接成功后，程序日志会显示：
   ```
   [info]: [ "[ws]", "ws client ready" ]
   ```

### B-5. 完成飞书平台事件订阅保存（管理员在 B-4 完成后操作）

> ⚠️ **此步骤只需管理员操作一次**，且必须在至少一台机器运行 ScreenDemo 并建立连接后执行。

1. 管理员确认某同事已完成 B-4 并看到连接成功日志
2. 立即回到飞书开放平台 → **事件与回调 → 回调配置**
3. 确认已选择长连接模式，已勾选 `im.message.receive_v1`
4. 点击「**保存**」
5. 保存成功提示出现即完成

## Part C：日常使用

### 截图快捷键

| 操作 | 快捷键（默认） |
|------|--------------|
| 截图并保存 | `Ctrl + Shift + S` |
| 打开 Memo 窗口 | 右键托盘 → Open Memo |
| 修改快捷键 | Settings → Shortcut |

### Memo 窗口功能

- **展开/收起**：点击左侧紫色把手
- **标记完成**：点击记录右侧的勾选按钮
- **编辑记录**：点击记录内容区域
- **推送所有未完成记录到飞书**：点击右上角 📤 按钮

### 飞书机器人使用

- 直接在飞书中向机器人发消息 → 消息自动保存为 Memo 记录
- 机器人会自动回复确认（若已启用）
- 定时（如早9晚6）机器人会主动推送当天未完成的记录

## Part D：常见问题

### Q: 安装后找不到托盘图标？
右下角任务栏 → 点击「^」展开隐藏图标 → 找到 ScreenDemo 图标

### Q: 快捷键与其他软件冲突？
Settings → Shortcut → 修改为其他组合键（如 `Ctrl+Alt+S`）

### Q: 飞书机器人无响应？
1. 确认 Target User ID 填写正确
2. 确认 App ID / App Secret 与管理员提供的一致
3. 重新保存设置，等待连接建立
4. 检查网络能否访问 `open.feishu.cn:443`

### Q: 数据保存在哪里？
默认保存在：`C:\Users\你的用户名\Pictures\ScreenDemo\`
可在 Settings → Custom Save Path 修改

### Q: 卸载后数据还在吗？
卸载时数据不会被删除，位于上述路径，可手动备份或删除
