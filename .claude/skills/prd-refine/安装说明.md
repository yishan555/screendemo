## ✨ 核心功能

### 1. 自动图片管理
- 从 PRD 中提取所有图片 URL
- 自动下载并根据上下文命名
- 保存到 `prdimages/` 文件夹
- 更新 PRD 中的引用为本地路径
- 支持断点续传和错误处理

### 2. 智能分析理解
- 深度理解业务背景和需求
- 识别核心功能和用户群体
- 发现不明确或缺失的需求
- 自动分组相关功能

### 3. 交互式澄清
- 结构化提问（每批最多 4 个问题）
- 提供上下文和选项
- 记录所有决策过程
- 支持多轮迭代

### 4. 场景拆分（Scene Breakdown）
将 PRD 分解为可实施的场景，每个场景包含：

- **场景概述**：ID、名称、描述、优先级
- **用户场景**：主流程、备选流程、边缘情况
- **功能需求**：编号、可测试、明确的需求
- **输入规范** ⭐：格式、来源、验证、示例
- **输出规范** ⭐：格式、状态、错误处理
- **数据需求**：实体、关系、约束
- **验收标准**：Given-When-Then 格式
- **依赖关系**：其他场景、外部系统
- **UI/UX 要求**：界面、交互、响应式
- **非功能需求**：性能、安全、可扩展性
- **错误处理**：边缘情况、异常场景

### 5. 生成完整文档
输出包括：
- `[原文件名]-refined.md` - 完善后的 PRD（开发者可直接使用）
- `prd-clarifications.md` - 所有问答记录
- `prd-refine-summary.md` - 处理摘要和指标
- `prdimages/` - 所有图片本地化
- `[原文件名].backup` - 原始 PRD 备份

## 💡 使用技巧

### 准备阶段
- 确保 PRD 有清晰的章节标题
- 包含原型图或线框图（URL 形式）
- 记录已知的约束条件

### 澄清阶段
- 认真回答问题 - 你的答案会塑造整个规范
- 提供具体示例
- 不确定时可以询问更多上下文
- 使用 "defer"（延迟）标记低优先级问题

### 审查阶段
- 逐场景审查精炼后的 PRD
- 运行验证脚本检查完整性
- 与技术团队分享
- 作为技术设计的基础

## 🆘 常见问题

**Q: 找不到 PRD 文件**
A: 提供完整路径或确保文件在当前目录

**Q: 图片下载失败**
A: 检查网络连接，某些网站可能阻止自动下载

**Q: 问题太多了**
A: 回答你能回答的，其他标记为 "defer"，skill 会做合理假设

**Q: 场景拆分太细/太粗**
A: 提供指导："创建更大的场景" 或 "拆分成更小的片段"
