# PRD Refine Skill 使用指南

## 快速开始

```bash
/prd-refine path/to/your-prd.md
```

## 主要改进（v1.1.0）

### 1. ✨ 新的输出目录结构

所有生成的文件现在统一放在 `newprd/` 目录下：


## 完整工作流程

### 步骤1：准备PRD文件

确保你的PRD文件是markdown格式（.md）

### 步骤2：运行Skill

```bash
/prd-refine 【DFX】DTS数据管理 PRD_副本.md
```

或者使用完整路径：
```bash
/prd-refine /path/to/【DFX】DTS数据管理 PRD_副本.md
```

### 步骤3：Skill自动执行以下操作

1. **定位PRD文件** - 找到指定的文件
2. **创建输出目录** - 创建 `newprd/` 和 `newprd/prdimages/`
3. **下载图片** - 自动下载所有图片（包括特殊格式URL）
4. **分析PRD内容** - 理解需求和业务背景
5. **提出澄清问题** - 交互式询问不明确的地方（每次最多4个问题）
6. **分解场景** - 将PRD分解为可实现的场景
7. **生成规格文档** - 创建详细的输入/输出规格和验收标准
8. **验证完整性** - 检查所有必需内容是否完整

### 步骤4：查看输出

```bash
# 查看生成的文件
ls -la newprd/

# 查看下载的图片
ls -la newprd/prdimages/

# 阅读精炼后的PRD
cat newprd/[原始文件名]-refined.md

# 查看澄清记录
cat newprd/prd-clarifications.md

# 查看处理摘要
cat newprd/prd-refine-summary.md
```

## 常见问题

### Q1: 图片下载失败怎么办？

**A**: 检查 `newprd/prd-refine-summary.md` 中的失败列表。常见原因：

1. **网络问题** - 检查网络连接
2. **需要认证** - 某些内部URL可能需要VPN或认证
3. **URL格式不支持** - 请反馈，我们会添加支持

手动下载失败的图片：
```bash
curl -L "失败的URL" -o newprd/prdimages/manual-image.png
```

### Q2: 如何找到我的PRD文件？

**A**: 如果只记得文件名的一部分：

```bash
# 搜索包含"DTS"的markdown文件
find . -name "*DTS*.md"

# 搜索包含"数据管理"的markdown文件
find . -name "*数据管理*.md"
```

### Q3: 澄清问题太多怎么办？

**A**:
- 一次最多4个问题
- 如果不确定，可以选择"defer"（延后）
- Skill会基于合理假设继续处理
- 可以在生成的文档中用 `[NEEDS CLARIFICATION]` 标记后续确认

### Q4: 场景分解太细或太粗怎么办？

**A**: 在运行时提供指导：

```bash
# 希望更大的场景
"创建更大的场景，涵盖完整的用户旅程"

# 希望更小的场景
"分解为更小、更专注的场景"
```

### Q5: 如何重新运行？

**A**:
- 可以多次运行以进一步精炼
- 每次运行会覆盖 `newprd/` 目录中的文件
- 如需保留之前的版本，先备份 `newprd/` 目录

## 输出文件详解

### 1. `[filename]-refined.md`
精炼后的完整PRD，包含：
- 场景分解
- 详细的输入/输出规格
- 验收标准
- 依赖关系
- 实现顺序建议

### 2. `prd-clarifications.md`
所有问答记录，包含：
- 提出的问题
- 用户的回答
- 决策依据

### 3. `prd-refine-summary.md`
处理摘要，包含：
- 图片下载统计
- 场景数量统计
- 复杂度评估
- 建议的实现顺序

### 4. `prdimages/`
所有下载的图片，命名规则：
```
[章节名称]-[描述].扩展名

例如：
- 用户流程-登录过程.png
- 架构-系统图.jpg
- 线框图-仪表板布局.png
```

## 最佳实践

### 1. PRD准备
- 尽可能详细
- 包含截图/原型图
- 明确业务背景
- 列出相关系统

### 2. 回答澄清问题
- 仔细阅读上下文
- 考虑技术可行性
- 明确优先级
- 记录假设

### 3. 检查输出
- 逐场景审查
- 验证输入/输出规格
- 确认验收标准可测试
- 检查依赖关系

### 4. 团队协作
- 与团队分享 `newprd/` 目录
- 使用 `prd-clarifications.md` 记录决策
- 基于 `prd-refine-summary.md` 规划实施

## 技术细节

### 支持的图片格式
- PNG (.png)
- JPEG (.jpg, .jpeg)
- GIF (.gif)
- WebP (.webp)
- SVG (.svg)
- BMP (.bmp)

### URL识别规则
1. 标准扩展名识别
2. 百度BOS特征识别（bcebos.com + /images/）
3. 内部服务识别（feishu-service, cfe-doc-backend）
4. 查询参数中的格式识别

### 下载策略
- 超时时间：30秒
- User-Agent：模拟浏览器
- 自动重试：否（避免过多请求）
- 错误处理：记录但不中断流程
